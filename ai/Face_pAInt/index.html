Face pAInt

Overview

The aim of this project was to create/assist with generative fine-art using computational photography techniques to extend a traditional studio shoot. This is achieved by training a diffusion model to understand the concept that an image/pattern which typically would appear as a background can be applied onto a face. For this to work, the model needs to learn the 3d semantic map of the face.

Images are the artefacts which can be a upscaled to high resolution as a final image or used as layers on top of the original image to maintain fine details with the options of further post processing. The goal is an aesthetic result intended for artistic display.

The motivation for this project was to create vibrant portraits using the look of paint on a face. Inspiration had come from images on-line; however, many of the most vibrant ones were clearly AI. I wanted to do this using real people. It reminded me of a photo shoot in the past with a make-up artist experimenting. I didn't know what would come out of the changing room. In post production I could do more, but I was limited to working on a 2d plane, where a true make-up artist worked on a 3d face.

This system touches on all of these points: intense face painting images, multiple random variations on a single pose and near pixel bound output so the output can be pulled into layers for further artistic expression maintaining the 3d feeling. The best post-production steps have been implemented in code and are part of the system after image generation.

Process

From a user's perspective, a 512x512 pixel portrait image is the primary input. To influence the face painting, a pattern picture along with a text prompt are specified. Inference parameters (guidance scales, etc) control how strong the face painting will be and how random the pattern or derivation from the original face structure.

The system employs a fine-tuned InstructPix2Pix (IP2P) UNet trained on a custom face-painting dataset to generate an image with the applied effect. At inference, the UNet operates within a Stable Diffusion 1.5 pipeline where it replaces the standard SD1.5 UNet. An image-prompt adapter is additionally attached to the pipeline to control the subtle balance between text prompt and the image. A parameter search grid was created to show the impact of the inference parameters.

Ultimately the goal was to achieve print level resolution. Rather than handling fine details employing a learned super-resolution model, the system transfers high-frequency detail from the original high-resolution photograph into the upscaled generated output via frequency separation. This was originally crafted in photo editing software, then codified for the notebook. This preserves authentic skin texture and photographic detail while retaining the generated paint structure, avoiding the risk of learned upscalers suppressing the stylistic modifications.

512x512 pixel image outputs come directly from the pipeline which can be displayed in the notebook or within a Gradio app where the user can experiment with the paint pattern influence via images and controls. Post-processing is included in the system which allows the user (both in the notebook and application) to see different combinations colour/luminosity in a parameter grid to experience different weights/moods of the effect.


* Photo qualit upscaling is currently only in the Colab cells, not the Gradio app. The postprocessing could also be achived in photo editing software if the user prefers.

Evaluation

The system has generative autonomy over paint-style makeup and its application to a portrait, within constraints that preserve photographic realism - motivated by compositing into editorial print, as if collaborating with an experimental MUA. Using the FACE model:
  • Eg: A portrait with a creative pattern applied.
  • Cg: A system generating a pattern and allowing adjustment before applying.
  • Ag: Colton's Creativity Tripod (skill, appreciation, imagination) -> Is the output worthy of printing/presenting to others/entertaining to look at?
  • Fg: A portrait image and pattern image influencing the design.

Value Added

Trained models

ControlNet

This was trained on 512x512px images and the corresponding depthmaps with the goal of maintaining pixel level accuracy in reconstruction. Later was discovered that Stable Diffusion ignores the custom ControlNet when using IPix2Pix. This was confirmed with experiments. On the upside, the base of Stable Diffusion 1.5 has training on faces and the three dimensional concept was essentially provided.
ControlNet model download (Google Drive)

InstructPix2Pix (IP2P)

To learn the effect of painting on a face, an InstructPix2Pix (IP2P) UNet model was trained. This would be used in Stable Diffusion 1.5 for execution. IP2P could be used efficiently without per inference fine tuning. During training, it was given a 512x512px face image without "paint", the painted face and a prompt which describe the image such as "An Indian woman with intricate face painting with some skin showing through." All of the face paint was applied via an opacity mask so that feathering of the texture along the hairline and on the neck would appear natural rather than an abrupt painted image. The output would be the generated 512x512px image.

Datasets

An additional value added component was a creation of datasets rendered in contolled lighting with a neutral grey background.

Dataset 1 - 3820 image sets

Each image set has there is the original 512x512 pixel portrait, a depth map and a painted face. The dataset has two diferent lighting setups (one the mirror of the other). The depth map was not used in this project; however, it would be useful for other training considering facial geometry.

Post Generation Processing

Post processing was first done in a paint program, but has now been moved into the notebook and mostly into the UX/app as well. By separating the output into colour and luminousity layers, some of the defects around the eyes are less visible. The generated layer output is then sandwiched between the low and high frequencies of the original image for full resolution detail.
The post processing is purely computational so the system doesn't require regeneration of the image/paint effect for adjustments to luminance, colour or sharpness.
